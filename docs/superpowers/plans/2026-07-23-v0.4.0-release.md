## 停止条件与恢复策略

- 任一完整验证或 dry-run 失败：不创建 tag、不推送、不发布，先修复根因并重新运行受影响检查。
- Git push 失败：保留本地发布提交和 tag，修复远程认证后重试同一 push，不重建 tag。
- tag 已推送但 npm 尚未发布：保留 tag，解决 npm 认证或权限后继续发布 `0.4.0`。
- SDK 发布成功、MCP 失败：只续发 MCP `0.4.0`；npm 版本不可撤回或覆盖。
- 发布后发现严重问题：停止推广并评估 `npm deprecate` 与后续修复版 `0.4.1`，不得重写 `v0.4.0` 或覆盖 npm `0.4.0`。
